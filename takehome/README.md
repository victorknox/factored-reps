# Mess3 Geometry Experiment

**Residual Stream Geometry in a Non-Ergodic Mess3 Mixture:**
Belief-State Representations Under Latent Component Uncertainty

This experiment trains a small transformer on next-token prediction over sequences
generated by a non-ergodic mixture of three distinct Mess3 (3-state HMM) processes,
then analyzes the geometry of its internal representations to understand how it
encodes hierarchical belief states.

> **Part of the [factored-reps](../) repository.** This directory depends on
> `fwh_core` (Mess3 HMM implementation) from the parent repo.

---

## Key Findings

| Experiment | Key Metric | Result |
|------------|-----------|--------|
| Hierarchical Posterior | Joint Y R² (10-fold CV) | **0.885 +/- 0.004** |
| Fractal Recovery | Per-component R² | **0.85 -- 0.95** |
| Subspace Orthogonality | comp-ID vs belief overlap | **< 0.01** |
| Emergence Transition | Block 0 MLP jump | R² 0.03 -> 0.43 |
| Dimensionality | k*_0.95 at final layer | **5 dims** (pos 14) |

The model achieves near-Bayes-optimal loss (1.0762 vs 1.076 nats) and discovers a
**surprisingly factored representation** where component identity and within-component
beliefs live in nearly orthogonal subspaces.

See [FINAL_REPORT.md](FINAL_REPORT.md) for the full write-up with figures.

## Reproduction

### Full Pipeline (one command)

```bash
cd factored-reps
bash takehome/reproduce.sh          # CPU (slow, ~2-3 hours)
bash takehome/reproduce.sh --gpu    # GPU  (recommended, ~30 min)
```

### Step-by-Step

All commands are run from the **repo root** (`factored-reps/`).

```bash
# 1. Generate dataset (50k train + 5k val sequences, ~2 min)
python takehome/data/generate_nonergodic_mess3.py

# 2. Train full transformer (200 epochs, ~10 min on GPU)
python takehome/train/train_transformer.py \
    --config takehome/configs/main_full.json \
    --data_dir takehome/results \
    --output_dir takehome/results/checkpoints_full

# 3. Baseline geometry analysis (PCA, probes, ~5 min)
python takehome/analysis/analyze_geometry.py \
    --checkpoint_dir takehome/results/checkpoints_full \
    --device cuda

# 4. Emergence analysis (layerwise heatmaps, ~3 min)
python takehome/analysis/extra_analysis.py \
    --checkpoint_dir takehome/results/checkpoints_full \
    --device cuda

# 5. Run experiments 1-5, 7 (main results, ~15 min)
python takehome/analysis/run_all_experiments.py \
    --checkpoint_dir takehome/results/checkpoints_full \
    --device cuda

# 6. Run experiment 6: K-scaling (trains 3 separate models, ~20 min on GPU)
python takehome/analysis/run_experiment6.py --device cuda
```

Replace `cuda` with `cpu` if no GPU is available (significantly slower).

### Smoke Test

Quick sanity check with a smaller model and fewer sequences:

```bash
python takehome/data/generate_nonergodic_mess3.py --n_train 1000 --n_val 200
python takehome/train/train_transformer.py \
    --config takehome/configs/smoke.json \
    --data_dir takehome/results \
    --output_dir takehome/results/checkpoints
```

## Experiment Overview

1. **Hierarchical Posterior Verification** -- 10-fold CV regression confirms joint posterior is linearly accessible (R² = 0.885)
2. **Dimensionality vs Context Position** -- PCA dimensionality grows from 2 to 5 as component identity resolves
3. **Component Separability** -- Sharp emergence at Block 0 MLP (R² jumps 0.03 -> 0.43)
4. **Geometry Visualization** -- 2D/3D PCA shows progressive component cloud separation
5. **Fractal Geometry Recovery** -- Linear regression recovers Mess3 fractal belief structure per component
6. **Dimensionality Scaling with K** -- Tests 3K-1 prediction across K=2,3,4
7. **Subspace Orthogonality** -- Component-ID and belief subspaces are nearly orthogonal (overlap < 0.01)
